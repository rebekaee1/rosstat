# ИИ-аналитика для SEO — пайплайн

**Источник:** правка `v2/edit_012` (видео НА правки 2). План, не код.

## Цель

Автоматически генерировать аналитические тексты под ключевые SEO-запросы (см. `docs/seo_long_tail_strategy.md`), публиковать их как статичные блоки на страницах индикаторов и/или отдельный раздел `/blog`. Цель — прирост long-tail трафика без ежемесячного ручного копирайтинга.

## Архитектура

```
[Cron job (06:30 МСК)]
    │
    ↓
[Trigger: после ETL CPI / key-rate / unemployment]
    │
    ↓
[generate_content.py]
    ├─ select indicator codes from priority list
    ├─ pull last 5y data from PostgreSQL
    ├─ build prompt: data + template + instructions
    ├─ call OpenAI (gpt-4o or gpt-5.x с reasoning)
    ↓
[review_queue (Postgres table `ai_content_drafts`)]
    │ (status: 'pending_review', 'approved', 'rejected')
    ↓
[Manual review UI / Telegram bot для approval]
    │
    ↓
[publish_to_db (`ai_content` table) + invalidate_cache]
    │
    ↓
[Frontend: блок «Аналитика» на странице индикатора]
```

## Компоненты

### 1. Backend

- Новая таблица `ai_content_drafts` (id, indicator_code, prompt_hash, draft_text, status, created_at, reviewed_at, reviewer).
- Новая таблица `ai_content` (id, indicator_code, slug, title, body_md, schema_org_jsonld, published_at, version).
- Сервис `app/services/ai_writer.py` с методами:
  - `prepare_prompt(indicator_code) → str`
  - `call_openai(prompt) → str`
  - `save_draft(indicator_code, text) → AiContentDraft`
- API `/api/v1/ai-content/{indicator_code}` (GET) — отдаёт опубликованный текст.

### 2. Шаблоны промптов

- `prompt_indicator_summary.md` (5 лет): «Сделай аналитический текст 800-1200 знаков о индикаторе X. Используй цифры [from], [to], [avg], [max]. Стиль: деловая аналитика, без воды.»
- `prompt_year_review.md` (фокус на конкретный год): «Опиши, как изменился индикатор X в YYYY году. Что повлияло. Сравни с предыдущим годом.»
- `prompt_forecast_explanation.md`: «Объясни прогноз индикатора X на следующие 12 месяцев. Не выдумывай факты вне данных.»

### 3. Frontend

- На странице индикатора (`IndicatorDetail.jsx`) ниже DataTable — секция «Аналитика», содержащая:
  - Заголовок (h2)
  - Текст в `<article>` с маркап ItemReviewed/Article schema.org
  - Дата публикации, badge «Сгенерировано ИИ»
- Если для индикатора нет опубликованного текста — секция не рендерится.

### 4. Review

- Telegram-бот `@forecast_econ_review_bot`: каждый новый draft отправляется в чат с кнопками «Approve / Reject / Edit».
- На approve — статус → `approved`, далее cron-job переносит в `ai_content` и инвалидирует кэш страницы.

## Стоимость

- Прайс gpt-4o ~ $5/1M input tokens, $15/1M output. Один длинный текст (1200 знаков) = ~600 input + ~400 output = $0.009.
- 84 индикатора × 1 раз в месяц = $0.76/мес.
- 3 шаблона промптов × ежемесячно = ~$2.5/мес.

## Риски

1. **Галлюцинации.** Mitigations: жёсткий promptu с цифрами в JSON, schema.org structured data сравнивается, отказываемся публиковать без human review.
2. **SEO-фильтр Яндекса** за «искусственный контент». Mitigations: каждый текст помечен `Generated by AI` (E-E-A-T), human-reviewed, привязан к структурированным данным (Dataset).
3. **Дубли.** Mitigations: temperature 0.7+ + версия (раз в 6 месяцев перегенерация).

## Этапы

1. **MVP (1-2 sprint)** — без UI ревью, только backend + Telegram approval. 5 индикаторов.
2. **Полный охват (3-4 sprint)** — все 84 индикатора, автогенерация раз в квартал.
3. **Расширение (5+ sprint)** — раздел `/blog` с тематическими статьями (например, «Как ключевая ставка влияет на инфляцию»).
