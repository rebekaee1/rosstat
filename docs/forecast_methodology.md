# Методология прогнозов инфляции — фиксация правил

Документ фиксирует правила прогнозов **по обратной связи пользователя (видео-ревью НА правки 3, апрель 2026)**. Часть пунктов отражает текущее поведение, часть помечена как TODO — требует доработки backend.

Источники прогнозных факторов:
- `Forecast.model_name = 'CPI-Monthly-*'` — месячные прогнозы ИПЦ (текущая модель Holt-Winters / SARIMAX);
- `Forecast.model_name = 'Inflation-12M-*'` — отдельная модель скользящей 12-месячной инфляции;
- См. `backend/app/services/forecaster.py` (`train_inflation_12m`).

## 1. Месячная инфляция (ИПЦ к предыдущему месяцу)

**Правила пользователя:** прогноз — ежемесячный, обновляется при выходе нового релиза ИПЦ Росстата.

**Текущее поведение:** так и работает. `forecasts.py /indicators/{code}/inflation` отдаёт `actuals + forecast`. Прогноз на 12 месяцев вперёд возвращается из БД.

**Статус:** ОК.

## 2. Квартальная инфляция

**Правила пользователя:**
- За **каждый прошедший квартал** — произведение трёх месячных ИПЦ за этот квартал минус 100 (фактическая квартальная инфляция).
- За **текущий незавершённый квартал** — комбинация: фактические значения уже закрытых месяцев + прогнозные оставшихся.
- За каждый из **4 будущих кварталов** — произведение трёх прогнозных месячных ИПЦ.
- Пересчёт **при каждом новом релизе ИПЦ** (когда становится известен новый месяц).

**Текущее поведение:**
- Серия `inflation-quarterly` строится в `calculation_engine.py::_compute_quarterly_inflation` как ленивый перерасчёт после ETL ИПЦ.
- В UI квартальная вкладка фильтруется на годовые блоки (см. `IndicatorDetail.jsx`, viewMode === 'quarterly').
- **Не реализовано:** прогнозные значения для квартальной серии не публикуются — `chartMode === 'quarterly' ? false` отключает прогноз.

## 3. Годовая инфляция (12-месячная скользящая)

**Правила пользователя:**
- Годовой прогноз — **агрегация из месячных прогнозов**, не отдельная модель. (Если есть выделенная модель — пусть остаётся, но она должна быть согласована с месячными прогнозами.)
- Текущий незавершённый год показывается как прогноз (не должно быть `«2025: 5.0%»` если 2025 ещё не закончился — должна быть метка «прогноз»).

**Текущее поведение:**
- Endpoint `/indicators/{code}/inflation` сначала проверяет `Inflation-12M-*` (выделенная модель), при отсутствии — fallback на агрегацию из месячного `CPI-Monthly-*`.
- Серия `inflation-annual` (для вкладки «Годовая») рассчитывается в `_compute_annual_inflation` как скользящее произведение 12 месячных ИПЦ.
- В UI (`IndicatorDetail.jsx`) вкладка «Годовая» фильтрует точки до 1 на год: `Map(year → последняя точка года)`.

## 4. Недельная инфляция

**Правила пользователя:**
- Прогнозы по неделям — **не делаем** (волатильно, неинформативно). В UI переключатель прогноза отключён для weekly mode (`forecastEnabled = false` при `viewMode === 'weekly'`).

**Текущее поведение:**
- В `IndicatorDetail.jsx` строка 519: `forecastEnabled = canForecast && viewMode !== 'quarterly' && viewMode !== 'annual' && viewMode !== 'weekly'` — переключатель грейдится, тултип «Прогноз недоступен для этого режима».

**Статус:** ОК.
